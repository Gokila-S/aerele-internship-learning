# Why OOP?

Imagine you're building a Student Management System.

Without OOP

```python
student_name = "Alice"
student_age = 20
student_marks = [90, 85, 95]
```

Now imagine managing 10,000 students.

You'll have thousands of variables.

Instead, group related data together.

```python
class Student:
    ...
```

Each student becomes an object.

---

# What is a Class?

A class is a **blueprint** for creating objects.

Think of it as an architectural drawing.

```
Blueprint

↓

House
```

Similarly,

```
Class

↓

Object
```

---

Example

```python
class Student:
    pass
```

No student exists yet.

Only the blueprint.

---

# What is an Object?

An object is an **instance** of a class.

```python
student = Student()
```

Now

```
student
```

is a real object.

You can create many objects.

```python
s1 = Student()

s2 = Student()

s3 = Student()
```

Each is independent.

---

# Class vs Object

| Class | Object |
|--------|---------|
| Blueprint | Actual instance |
| Defines structure | Holds data |
| Created once | Can create many |

Example

```
Student

↓

Alice

Bob

Charlie
```

One class.

Many objects.

---

# Attributes

Attributes store data.

Example

```python
class Student:

    def __init__(self, name, age):

        self.name = name

        self.age = age
```

Usage

```python
student = Student("Alice", 20)

print(student.name)
```

Output

```
Alice
```

---

## Instance Attributes

Each object has its own values.

```python
s1 = Student("Alice",20)

s2 = Student("Bob",21)
```

```
s1.name

↓

Alice

s2.name

↓

Bob
```

Independent.

---

# Methods

Methods are functions inside a class.

Example

```python
class Student:

    def greet(self):
        print("Hello")
```

Call

```python
student.greet()
```

Output

```
Hello
```

---

# Understanding `self`

Most confusing topic for beginners.

`self` refers to **the current object**.

Example

```python
class Student:

    def show(self):
        print(self.name)
```

When

```python
alice.show()
```

Python internally does

```python
Student.show(alice)
```

So

```
self

↓

alice
```

---

## Why is `self` Needed?

Without it,

Python wouldn't know which object's data to use.

Example

```python
alice.name

↓

Alice

bob.name

↓

Bob
```

`self` tells Python which object is calling the method.

---

# Constructor (`__init__`)

Called automatically when an object is created.

Example

```python
class Student:

    def __init__(self, name):

        self.name = name
```

Usage

```python
student = Student("Alice")
```

Python automatically executes

```python
__init__()
```

---

# Why Use Constructors?

Initialize object state.

Instead of

```python
student = Student()

student.name = "Alice"

student.age = 20
```

Do

```python
student = Student("Alice",20)
```

Cleaner.

---

# Instance Variables vs Class Variables

## Instance Variable

Belongs to each object.

```python
class Student:

    def __init__(self,name):

        self.name = name
```

Each object has its own name.

---

## Class Variable

Shared by all objects.

```python
class Student:

    college = "KEC"
```

Every student shares

```
KEC
```

Example

```python
print(Student.college)
```

or

```python
print(student.college)
```

---

## When to Use Class Variables?

Shared information.

Examples

- College name
- Tax rate
- Company name
- Configuration

Avoid storing object-specific data in class variables.

---

# Encapsulation

Encapsulation means **keeping related data and behavior together**.

Instead of

```python
student_name

student_marks

calculate_grade()
```

Spread everywhere,

Keep them together.

```python
class Student:

    marks

    calculate_grade()
```

Everything related stays inside the class.

---

# Why Encapsulation?

- Easier maintenance
- Better organization
- Less accidental modification
- Cleaner code

---

# Access Modifiers (Python Style)

Python doesn't have true private variables.

Instead it follows conventions.

Public

```python
name
```

Protected (convention)

```python
_name
```

Private (name mangling)

```python
__name
```

---

Example

```python
class Student:

    def __init__(self):

        self.__password = "123"
```

Outside

```python
student.__password
```

Fails.

Python changes the internal name.

---

# Should Everything Be Private?

No.

Python trusts developers.

Use private variables only when hiding implementation details.

---

# Object Lifecycle

```
Create Object

↓

__init__()

↓

Use Methods

↓

Object Deleted

↓

Garbage Collector Frees Memory
```

Python automatically manages memory.

---

# Code Review Example

Review

```python
class Student:

    pass

student = Student()

student.name = "Alice"

student.age = 20
```

Problem

Object created in an incomplete state.

Better

```python
class Student:

    def __init__(self,name,age):

        self.name = name

        self.age = age
```

Now every student has valid data.

# Common Beginner Mistakes

❌ Forgetting `self`.

❌ Modifying attributes outside the class unnecessarily.

❌ Using class variables for object-specific data.

❌ Creating empty objects and assigning values later.

❌ Confusing classes with objects.
