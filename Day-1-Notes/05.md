# Topic 10 – Virtual Environments (`venv`)

A **virtual environment** is an isolated Python environment with its own interpreter, installed packages, and dependencies.

Without virtual environments, all projects share the same packages, causing dependency conflicts.

---

## Why do we need Virtual Environments?

Imagine two projects.

```
Project A
requests==2.31

Project B
requests==2.26
```

If both use the global Python installation, one project may break when the package version changes.

Virtual environments solve this by isolating each project.

```
System Python
│
├── Project A
│      ├── venv
│      └── requests 2.31
│
└── Project B
       ├── venv
       └── requests 2.26
```

Each project has its own packages.

---

## Create a Virtual Environment

```bash
python -m venv .venv
```

- `python` → Python interpreter
- `-m` → Run a module
- `venv` → Virtual environment module
- `.venv` → Environment folder

---

## Activate

### Windows

```bash
.venv\Scripts\activate
```

### Linux / macOS

```bash
source .venv/bin/activate
```

Prompt changes

```
(.venv)
```

meaning the environment is active.

---

## Deactivate

```bash
deactivate
```

Returns to system Python.

---

## Verify

```bash
where python
```

or

```bash
which python
```

Should point inside

```
.venv
```

---

## Common Mistakes

❌ Installing packages before activating venv.

❌ Creating one virtual environment for every assignment instead of one per project.

❌ Committing `.venv` to Git.

Add

```
.venv/
```

to `.gitignore`.

# Topic 11 – pip

`pip` is Python's package manager.

It downloads packages from **PyPI (Python Package Index)**.

---

```bash

```

---

## Install Specific Version

```bash

```

---

## Upgrade

```bash

```

---

## Remove

```bash
pip uninstall requests
```

---

## View Installed Packages

```bash
pip list
```

---

## Show Package Details

```bash
pip show requests
```

Useful information

# Topic 12 – requirements.txt

A project should always record its dependencies.

Example

```
requests==2.31.0
Flask==3.0.2
pandas==2.2.2
```

---

## Generate

```bash
pip freeze > requirements.txt
```

---

## Install

```bash
pip install -r requirements.txt
```

Every developer now gets the same package versions.

---

## Why is this important?

Without it,

Developer A

```
numpy 2.1
```

Developer B

```
numpy 1.24
```

Application behaves differently.

With

```
requirements.txt
```

everyone works with identical versions.

---

## Difference

```
pip list
```

Human readable.

```
pip freeze
```

Machine readable.

Use

```
pip freeze
```

to create

```
requirements.txt
```

# Topic 13 – pyproject.toml

Modern Python projects use

```
pyproject.toml
```

instead of scattered configuration files.

It can store

- Project metadata
- Dependencies
- Build configuration
- Tool configuration

Example

```toml
[project]

name = "student-manager"

version = "1.0.0"

dependencies = [
    "requests",
    "flask"
]
```

Many tools read configuration from this file.

Examples

- Ruff
- Black
- Mypy
- Poetry
- Hatch

---

## Why is it useful?

One configuration file.

Less duplication.

Cleaner projects.

# Day 1 Code Review Exercise

Review the following code.

```python
file = open("students.txt")

students = []

for i in range(len(file.readlines())):
    students.append(file.readlines()[i].strip())

file.close()

print(students)
```

---

## Problems

1. File opened manually.
2. `close()` may never execute.
3. `readlines()` called repeatedly.
4. Index loop unnecessary.
5. Not Pythonic.
6. Inefficient.

---

## Improved Version

```python
with open("students.txt") as file:
    students = [line.strip() for line in file]

print(students)
```

Improvements

- Automatic cleanup
- Single file read
- List comprehension
- Cleaner
- Faster
