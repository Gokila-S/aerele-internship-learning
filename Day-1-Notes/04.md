# Topic 9 – Pythonic Idioms

Python is expressive. Instead of writing everything manually, it provides built-in constructs that make code shorter, clearer, and less error-prone.

**Rule:** Prefer the Pythonic way unless readability suffers.

---

# List Comprehensions

## Why?

Transform or filter collections in one readable expression.

Instead of:

```python
squares = []

for i in range(5):
    squares.append(i * i)
```

Use

```python
squares = [i * i for i in range(5)]
```

Output

```python
[0, 1, 4, 9, 16]
```

---

## Syntax

```python
[new_value for item in iterable]
```

Filtering

```python
evens = [n for n in numbers if n % 2 == 0]
```

---

## Nested Comprehension

```python
matrix = [
    [1,2],
    [3,4]
]

flatten = [x for row in matrix for x in row]
```

Output

```python
[1,2,3,4]
```

---

## When NOT to use

Avoid if it becomes difficult to read.

Bad

```python
result = [
    ...
    ...
    ...
]
```

If the expression spans multiple conditions and nested loops, use a normal loop.

Readability > Shortness.

---

## Common Mistakes

❌ Writing everything as a comprehension.

Only use them when they improve clarity.

---

# enumerate()

## Problem

Need both index and value.

Many beginners write

```python
for i in range(len(names)):
    print(i, names[i])
```

Pythonic

```python
for index, name in enumerate(names):
    print(index, name)
```

Output

```
0 Alice
1 Bob
2 Charlie
```

---

## Why use enumerate?

- Cleaner
- No indexing mistakes
- More readable

---

## Common Mistake

Don't manually maintain a counter.

```python
count = 0

for item in items:
    ...
    count += 1
```

Use

```python
enumerate(items)
```

---

# zip()

## Problem

Need to iterate over multiple lists.

Instead of

```python
for i in range(len(names)):
    print(names[i], marks[i])
```

Use

```python
for name, mark in zip(names, marks):
    print(name, mark)
```

Output

```
Alice 90
Bob 85
Charlie 95
```

---

## Multiple Iterables

```python
for a, b, c in zip(x, y, z):
    ...
```

---

## Common Mistake

Using index-based loops when the position isn't needed.

---

# Tuple Unpacking

Python allows unpacking directly.

Instead of

```python
student = ("John", 21)

name = student[0]
age = student[1]
```

Use

```python
name, age = student
```

Cleaner.

---

## Swapping Variables

Without temp variable

```python
a, b = b, a
```

One of Python's nicest features.

---

## Ignoring Values

```python
name, _, age = data
```

Convention

```
_
```

means

"I don't need this value."

---

# f-Strings

Introduced in Python 3.6.

Preferred string formatting method.

Instead of

```python
print("Hello " + name)
```

or

```python
print("Hello {}".format(name))
```

Use

```python
print(f"Hello {name}")
```

---

## Expression Support

```python
a = 10
b = 20

print(f"Sum = {a+b}")
```

Output

```
Sum = 30
```

---

## Formatting

```python
price = 99.4567

print(f"{price:.2f}")
```

Output

```
99.46
```

---

## Why?

- Fast
- Readable
- Recommended

# EAFP vs LBYL

One of Python's biggest mindset changes.

---

## LBYL

Look Before You Leap

Check everything first.

Example

```python
if key in student:
    print(student[key])
```

---

## EAFP

Easier to Ask Forgiveness than Permission

Attempt.

Handle failure.

```python
try:
    print(student[key])
except KeyError:
    print("Not Found")
```

---

## Why Python prefers EAFP

Objects may change between checking and using.

Checking twice is unnecessary.

Handling exceptions is often cleaner.

---

## When to use LBYL

Use when exceptions are expensive or failure is expected frequently.

Example

```python
if file_exists:
```

---

## When to use EAFP

Most object operations.

Dictionary lookup.

Attribute access.

Parsing.

---

# Context Managers (`with`)

Probably the most important Python feature after functions.

---

## Problem

Resources must always be released.

Example

```python
file = open("data.txt")

content = file.read()

file.close()
```

What if

```python
file.read()
```

raises an exception?

```
close()
```

never executes.

Resource leak.

---

## Solution

```python
with open("data.txt") as file:
    content = file.read()
```

Python guarantees

```
file.close()
```

runs automatically.

Even if an exception occurs.

---

## Mental Model

```
Enter

↓

Do Work

↓

Exit Automatically
```

---

## Common Uses

Files

```python
with open(...)
```

Database Connections

```python
with connection:
```

Locks

```python
with lock:
```

Temporary Files

```python
with tempfile
```

---

## Benefits

- Cleaner
- Safer
- Automatic cleanup

---

## Common Mistakes

❌

```python
open()

...

close()
```

Always prefer

```python
with
```

unless absolutely necessary.

# Code Review Exercise

Improve this code.

```python
file = open("marks.txt")

for i in range(len(students)):
    print(students[i])

file.close()
```

Possible improvements

- Use `with`
- Use `enumerate()` only if index is required
- Remove index loop if unnecessary

Better

```python
with open("marks.txt") as file:
    for student in students:
        print(student)
```

# Key Takeaways

✅ Prefer readability over cleverness.

✅ Use comprehensions for simple transformations.

✅ Use `enumerate()` instead of `range(len())`.

✅ Use `zip()` when iterating over multiple iterables.

✅ Use tuple unpacking instead of indexing.

✅ Prefer f-strings for formatting.

✅ Understand EAFP vs LBYL.

✅ Always use `with` for resource management.
