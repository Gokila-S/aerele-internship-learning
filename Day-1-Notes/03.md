# Topic 5 – `is` vs `==`

This is one of the most common Python interview questions.

---

## Core Idea

| Operator | Checks |
|----------|--------|
| `==` | Value Equality |
| `is` | Object Identity (same object in memory) |

---

## Mental Model

Think of two notebooks.

Notebook A

```
Math Notes
```

Notebook B

```
Math Notes
```

Contents are identical.

But they are two different notebooks.

```
Notebook A == Notebook B      ✅ True
Notebook A is Notebook B      ❌ False
```

---

## Example 1

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)
print(a is b)
```

Output

```python
True
False
```

Why?

```
a ---> [1,2,3]

b ---> [1,2,3]
```

Different objects

Same values.

---

## Example 2

```python
a = [1,2,3]
b = a

print(a == b)
print(a is b)
```

Output

```python
True
True
```

Both variables point to the same object.

---

## When should `is` be used?

Almost never.

Main use case:

```python
if value is None:
```

Instead of

```python
if value == None:
```

Why?

There is only one `None` object.

Identity check is the correct semantic.

---

## Common Mistakes

❌

```python
if x == None:
```

✅

```python
if x is None:
```

---

## Interview Question

Why not

```python
a is 100
```

To compare numbers?

Because `is` checks identity, not value.

Always use

```python
a == 100
```

---

## LLM Review

If ChatGPT generates

```python
if user == None:
```

Change it to

```python
if user is None:
```

# Topic 6 – Dynamic Typing

---

## What is Dynamic Typing?

Variables do **not** have fixed types.

Objects have types.

Variables simply reference objects.

---

Example

```python
x = 10
```

```
x

↓

int object
```

Later

```python
x = "Hello"
```

Now

```
x

↓

str object
```

Nothing wrong.

Variable changed its reference.

---

## Compare with Java

Java

```java
int x = 10;

x = "Hello";
```

Compile Error.

Python

```python
x = 10
x = "Hello"
```

Perfectly valid.

---

## Why Python chose this?

Flexibility.

Less boilerplate.

Rapid development.

---

## Does Dynamic Typing mean "no types"?

No.

Every object has a type.

```python
print(type(10))
print(type("abc"))
```

Output

```python
<class 'int'>
<class 'str'>
```

Types exist.

Variables simply don't own one forever.

---

## Advantages

- Flexible
- Less code
- Faster development
- Easy prototyping

---

## Disadvantages

Errors occur at runtime.

Example

```python
x = 10

x.append(5)
```

Runtime error.

---

## How modern Python solves this?

Type Hints

```python
def add(a: int, b: int) -> int:
    return a + b
```

Then tools like

- mypy
- Pyright

catch mistakes before execution.

---

## Common Mistakes

Thinking

```
Variables have types.
```

Wrong.

Objects have types.

Variables reference objects.

---

## Interview Question

Difference between Static and Dynamic Typing?

Static

- Variable type fixed.
- Compile-time checking.

Dynamic

- Variable can reference different object types.
- Errors appear during execution unless checked by tooling.

# Topic 7 – Duck Typing

One of Python's defining philosophies.

---

## Definition

> "If it walks like a duck and quacks like a duck, treat it like a duck."

Python cares about **behavior**, not inheritance.

---

Instead of asking

```
Is this a Bird?
```

Python asks

```
Can it fly?
```

---

## Example

```python
class Dog:
    def speak(self):
        print("Bark")


class Cat:
    def speak(self):
        print("Meow")
```

Function

```python
def make_sound(animal):
    animal.speak()
```

Works for

```python
Dog()

Cat()
```

No inheritance required.

---

## Why?

Because both provide

```
speak()
```

Python only needs the required behavior.

---

## Java Mindset

Check type.

```java
instanceof
```

Python Mindset

Just use it.

If it fails,

handle the exception.

---

## Benefits

- Flexible
- Less coupling
- Easier testing
- Cleaner APIs

---

## Real Example

File object

```python
file.read()
```

StringIO

```python
buffer.read()
```

Network Stream

```python
socket.read()
```

Same interface.

Different implementations.

---

## LLM Review

Avoid

```python
if isinstance(obj, Student):
```

when all you need is

```python
obj.get_name()
```

Check behavior, not concrete type.

# Topic 8 – Truthiness

Python allows objects to behave like booleans.

---

Instead of

```python
if len(items) > 0:
```

write

```python
if items:
```

---

Falsy Values

```python
False
None
0
0.0
''
[]
{}
set()
```

Everything else is Truthy.

---

## Examples

```python
name = ""

if name:
    print("Exists")
```

Nothing prints.

---

```python
numbers = [1]

if numbers:
    print("Not Empty")
```

Output

```
Not Empty
```

---

## Why use Truthiness?

Cleaner.

Readable.

Idiomatic Python.

---

## Common Mistakes

❌

```python
if len(items) != 0:
```

✅

```python
if items:
```

---

❌

```python
if len(items) == 0:
```

✅

```python
if not items:
```

---

## Interview Question

Which is more Pythonic?

```python
if len(users) > 0
```

or

```python
if users
```

Answer

Second.

Because Python objects already know whether they're empty.

## Common Beginner Mistakes

- Using `is` for value comparison
- Using `== None`
- Thinking variables own values
- Confusing mutable with immutable
- Excessive `isinstance()` checks
- Writing `if len(list) > 0`
- Assuming dynamic typing means "no types"

---

## Code Review Checklist

Before approving code ask:

- Am I comparing identity or value?
- Is `None` checked with `is`?
- Is unnecessary type checking used?
- Can truthiness simplify this?
- Is the code relying on behavior rather than concrete types?
- Am I accidentally modifying a shared mutable object?
