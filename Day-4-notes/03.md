Internal helper.

# `__all__`

Controls what gets exported.

Example

```python
__all__ = [
    "calculate_total",
    "Invoice"
]
```

Only these names are exported when someone writes

```python
from package import *
```

Even though wildcard imports are discouraged, `__all__` defines the intended public API.

# Interview Questions

### What is packaging?

Organizing Python code into reusable, installable units.

---

### What is `pyproject.toml`?

The central configuration file for modern Python projects. It stores project metadata, dependencies, and tool configurations.

---

### Explain Semantic Versioning.

`MAJOR.MINOR.PATCH`

- Major â†’ Breaking changes
- Minor â†’ New features
- Patch â†’ Bug fixes

---

### What does `pip install -e .` do?

Installs the project in editable mode so code changes are immediately reflected without reinstalling.

---

### What is coupling?

The level of dependency between modules.

Lower coupling is better.

---

### What is cohesion?

How closely related the responsibilities inside a module are.

Higher cohesion is better.

---

### Why keep tests in a separate folder?

To isolate production code from test code and improve maintainability.
