# What is a Module?

A **module** is simply a Python file.

Example

```
math_utils.py
```

Inside

```python
def add(a, b):
    return a + b
```

Another file

```python
import math_utils

print(math_utils.add(2,3))
```

Output

```
5
```

One file = One module.

# What is a Package?

A package is a folder containing related modules.

Example

```
project/

    utils/

        math.py

        string.py

        date.py
```

```
utils
```

is the package.

```
math.py
```

is a module.

---

# `__init__.py`

Traditionally, a package contains

```
__init__.py
```

Example

```
utils/

    __init__.py

    math.py

    string.py
```

Originally it marked the directory as a package.

Modern Python can work without it in many cases, but most professional projects still include it.

# Importing Modules

Import entire module

```python
import math
```

Usage

```python
math.sqrt(25)
```

---

Import specific function

```python
from math import sqrt

print(sqrt(25))
```

---

Import multiple

```python
from math import sin, cos
```

---

Import with alias

```python
import numpy as np
```

Very common.

---

# Which Import Style is Best?

Usually

```python
import module
```

is clearer.

Example

```python
math.sqrt()
```

instead of

```python
sqrt()
```

The source is obvious.

# Avoid Wildcard Imports

Bad

```python
from math import *
```

Why?

You don't know

- where functions came from
- which names already exist

Avoid in production code.

# Module Search Path

When Python imports

```python

```

it searches

1. Current folder

↓

2. Installed packages

↓

3. Standard Library

```

```

# Avoid Circular Imports

One of the most common project problems.

Bad

```
student.py

↓

imports

↓

teacher.py

↓

imports

↓

student.py
```

Infinite dependency.

Python cannot finish importing.

---

## Solution

Move shared logic.

Example

```
student.py

teacher.py

↓

common.py
```

Both import

```
common.py
```

instead of each other.

# `if __name__ == "__main__"`

A very common interview topic.

Suppose

```python
def hello():
    print("Hello")
```

You also write

```python
hello()
```

Now importing this module executes it automatically.

Sometimes you don't want that.

Instead

```python
def hello():
    print("Hello")

if __name__ == "__main__":
    hello()
```

Now

Running directly

```
python file.py
```

executes it.

Importing

```python
import file
```

does not.

---

# Why is this Useful?

Useful for

- Testing
- Demo code
- Scripts

Without affecting imports.

# Interview Questions

### What is a module?

A single Python file.

---

### What is a package?

A directory containing related modules.

---

### Difference?

Module

```
math.py
```

Package

```
utils/

    math.py
```

---

### Why use packages?

- Better organization
- Easier maintenance
- Code reuse
- Scalability

---

### Why avoid wildcard imports?

They pollute the namespace and make code harder to read.

---

### Purpose of `if __name__ == "__main__"`?

Run code only when the file is executed directly, not when imported.

# Common Beginner Mistakes

❌ Keeping everything in one file.

❌ Using wildcard imports.

❌ Creating circular imports.

❌ Mixing unrelated functionality inside one module.

❌ Using unclear package structures.

# LLM Code Review Checklist

Whenever ChatGPT generates a project, ask:

- Are modules split logically?
- Is each module responsible for one thing?
- Are imports clear?
- Can wildcard imports be removed?
- Are circular dependencies possible?
- Would another developer find the file easily?
