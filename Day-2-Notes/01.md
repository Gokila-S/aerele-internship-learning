# Why Functions Matter

Imagine writing the same code in five different places.

```python
total = marks1 + marks2 + marks3
average = total / 3
```

Later your teacher says,

> "Calculate the average of 5 subjects."

Now you need to modify all five places.

Instead, write it once.

```python
def average(marks):
    return sum(marks) / len(marks)
```

Now everyone uses the same function.

Functions help us:

- Reuse code
- Reduce bugs
- Improve readability
- Make testing easier
- Hide implementation details

# What is a Function?

A function is a **named block of reusable code** that performs one specific task.

Think of it like a machine.

```
Input
   ↓
Function
   ↓
Output
```

Example

```python
def square(n):
    return n * n
```

Calling it

```python
print(square(5))
```

Output

```
25
```

# Parameters vs Arguments

One of the most common interview questions.

## Parameters

Variables defined in the function.

```python
def greet(name):
```

Here,

```
name
```

is a parameter.

---

## Arguments

Actual values passed while calling.

```python
greet("Alice")
```

```
"Alice"
```

is an argument.

---

## Easy Way to Remember

```
Definition → Parameters

Call → Arguments
```

---

Example

```python
def multiply(a, b):
    return a * b

multiply(5, 8)
```

Parameters

```
a
b
```

Arguments

```
5
8
```

# Return vs Print

Many beginners confuse these.

## Print

Displays something.

```python
def add(a, b):
    print(a + b)
```

Output

```
5
```

But

```python
x = add(2, 3)

print(x)
```

Output

```
5
None
```

Because

```
print()
```

doesn't return anything.

---

## Return

Returns the value.

```python
def add(a, b):
    return a + b
```

Now

```python
x = add(2, 3)

print(x)
```

Output

```
5
```

---

## Rule

Use

```
return
```

when another function or caller needs the result.

Use

```
print()
```

only for displaying information to users.

# Single Responsibility Principle (SRP)

A function should do **one thing well**.

Bad

```python
def register_user(user):
    save_to_database(user)
    send_email(user)
    print(user)
    generate_pdf(user)
```

This function has multiple responsibilities.

- Database
- Email
- Printing
- PDF

If email changes,

this function changes.

If PDF changes,

this function changes.

Hard to maintain.

---

## Better Design

```python
def save_user(user):
    ...
```

```python
def send_email(user):
    ...
```

```python
def generate_pdf(user):
    ...
```

```python
def register_user(user):
    save_user(user)
    send_email(user)
    generate_pdf(user)
```

Each function has one responsibility.

---

## Benefits

- Easier testing
- Easier debugging
- Easier reuse
- Easier maintenance

# Pure Functions

A pure function

- Depends only on inputs.
- Doesn't modify outside data.
- Always produces the same output.

Example

```python
def square(x):
    return x * x
```

```
square(5)
```

Always

```
25
```

---

Another example

```python
def add(a, b):
    return a + b
```

Very predictable.

---

## Benefits

- Easy testing
- Easy debugging
- Safe to reuse
- No hidden surprises

---

# Impure Functions (Side Effects)

A side effect means the function changes something outside itself.

Example

```python
students = []

def add_student(name):
    students.append(name)
```

This modifies global data.

---

Another example

```python
def save():
    with open("marks.txt", "w") as file:
        file.write("100")
```

Changes a file.

---

Other side effects

- Writing files
- Sending emails
- Database updates
- Network requests
- Printing
- Modifying globals

---

## Should We Avoid Side Effects?

No.

Real applications need them.

But keep them separate from business logic whenever possible.

Example

Instead of

```python
def calculate_total(cart):
    total = sum(cart)
    print(total)
    save_to_database(total)
    return total
```

Prefer

```python
def calculate_total(cart):
    return sum(cart)
```

Then

```python
total = calculate_total(cart)

save_to_database(total)

print(total)
```

Now the calculation function is reusable and easy to test.

# Local vs Global Variables

## Local

Created inside a function.

```python
def demo():
    x = 10
```

Exists only inside

```
demo()
```

---

## Global

Created outside functions.

```python
count = 0
```

Accessible everywhere.

---

Avoid modifying globals.

Bad

```python
count = 0

def increment():
    global count
    count += 1
```

Global state makes debugging difficult.

---

Better

```python
def increment(count):
    return count + 1
```

---

# Code Review Example

Review this function.

```python
students = []

def register(name):
    students.append(name)
    print(name)
    return students
```

Problems

- Modifies global state.
- Prints and returns.
- Multiple responsibilities.
- Difficult to test.

Better

```python
def register(students, name):
    students.append(name)
    return students
```

Printing should happen outside.

### Difference between Parameter and Argument?

Parameter

Variable in function definition.

Argument

Actual value passed during function call.

---

### Difference between Return and Print?

Print

Displays output.

Return

Sends a value back to the caller.

---

### What is a Pure Function?

A function with no side effects that depends only on its inputs.

---

### What is a Side Effect?

Any operation that changes external state, such as:

- Modifying globals
- Writing files
- Sending emails
- Database operations
- Printing
- Network requests

---

### What is SRP?

A function should have **one reason to change**.

# Common Beginner Mistakes

❌ Using `print()` instead of `return`.

❌ Writing 300-line functions.

❌ Using meaningless names like `abc()` or `func1()`.

❌ Modifying global variables unnecessarily.

❌ Mixing business logic with input/output.

❌ Creating functions that perform multiple unrelated tasks.

---

# LLM Code Review Checklist

Whenever ChatGPT generates a function, ask:

- Does it have a single responsibility?
- Is the function name descriptive?
- Should it return instead of print?
- Does it unnecessarily modify global state?
- Can side effects be separated from business logic?
- Is the function small and readable?
- Can it be tested independently?
